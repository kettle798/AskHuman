# 开发计划：项目多语言（i18n）

> 关联需求：`docs/specs/i18n.md`
> 计划描述方案与技术/规则细节，具体代码以实现为准。

## 0. 方案总览

```
单一配置 config.general.language = auto | en | zh
        │
        ├── 前端：vue-i18n（en/zh 词条）
        │     启动读 config → 解析 locale(auto→navigator.language)
        │     监听 settings-updated → 实时切 locale
        │     GUI 内后端文案：speech-* 事件发「语义 key(+参数)」→ t() 翻译
        │
        └── 后端(Rust)：i18n 模块（Lang{En,Zh} + tr(key)）
              解析 locale(auto→sys-locale)
              用于 CLI / 窗口标题 / macOS 菜单·Dock / 通知 / 远程渠道
              save_settings 检测语言变化 → 重建菜单 + 更新已开窗口标题
              prompts.cli_reference() 固定英文
```

源语言=英文：所有 key 以英文为基准，`zh` 为翻译；缺失回退 `en`。

---

## 1. 配置与依赖

- `config.rs`：`GeneralConfig` 增 `language: String`（默认 `"auto"`），更新 `Default` 与测试。
- `types.ts`：`GeneralConfig` 增 `language: "auto" | "en" | "zh"`（或 `string`）。
- 依赖：前端 `vue-i18n`；后端 `sys-locale`（用包管理器装最新版，不手填版本）。
- locale 解析规则（前后端一致，D10）：显式 `en`/`zh` 直用；`auto`→取系统首选语言，`zh*`→`zh`，否则 `en`。

## 2. 前端（vue-i18n）

### 2.1 框架
- 新增 `src/i18n/index.ts`：`createI18n({ legacy:false, locale, fallbackLocale:'en', messages:{en,zh} })`。
- 词条 `src/i18n/en.ts`、`src/i18n/zh.ts`，按命名空间组织：`common`、`popup`、`settings`、`speech`、`shortcut` 等。
- `main.ts`：bootstrap 时 `getSettings()` 取 `language` → 解析 locale → `app.use(i18n)`；解析前可先用 `navigator.language` 兜底，拿到配置再校正。

### 2.2 实时切换
- `PopupView`/`SettingsView` 已监听/将监听 `settings-updated`：收到后 `i18n.global.locale.value = resolve(payload.language)`。
- `SettingsView` 改语言时：本窗口立即切 locale + `persist()`；其它窗口经事件切换。

### 2.3 文案抽取
- `App.vue`、`PopupView.vue`、`SettingsView.vue`：所有可见字符串改 `t('ns.key')`（含 placeholder、title、aria-label、按钮、确认弹层、文件 chip、状态/错误行、tab、card 标题/描述等）。
- `src/lib/shortcut.ts`（D13）：`shortcutConflict` 返回 `{ key, params }`（或 key 字符串）；`formatShortcut` 的“无”由调用方处理。`SettingsView` 用 `t()` 渲染冲突提示与“无”。
- `src/lib/theme.ts`：可见错误（若有）走 i18n 或保留内部日志（非可见则不动）。

### 2.4 GUI 内后端文案（语音）
- 监听 `speech-status`/`speech-error` 的 handler：payload 改为**语义 key（+参数）**，用 `t('speech.'+key, params)` 显示。
- 定义 `speech.*` 词条（见 §3 的 key 集）。

## 3. GUI 内后端事件 → 语义 key（语音）

- Swift `SpeechEngine`/`SpeechBridge`：`onStatus`/`onError` 改为输出**稳定 key 字符串**（必要时附参数，如 locale）。建议：
  - 状态 key：`preparing`、`listening`、`downloadingModel`、`modelReady`。
  - 错误 key：`needMacos26`、`unsupportedLocale`(参数 locale)、`noAudioFormat`、`authDenied`、`bridgeNotReady`；系统 `NSError` 无法归类者 → key `generic` 并附原始 message 作为参数。
- `speech.rs`：事件 payload 由「人类文案」改为「key（+参数）」结构（如 `{ key, params }` 或 `key|param`）。前端按约定解析。
- 前端 `speech.*` 词条覆盖上述 key；`generic` 模板含 `{message}`。

## 4. 后端 Rust i18n（CLI / 标题 / 菜单 / 通知 / 渠道）

### 4.1 i18n 模块
- 新增 `src-tauri/src/i18n.rs`：`enum Lang { En, Zh }`、`fn resolve(cfg_language:&str)->Lang`（auto→`sys_locale::get_locale()`）、`fn tr(lang, key, args)->String`（match 分区组织：`cli`、`title`、`menu`、`channel`、…）。
- 取语言：用户可见入口处 `Lang::resolve(&AppConfig::load().general.language)`。

### 4.2 CLI（D6，全本地化）
- `cli/help.rs`：`help_text(lang)`、`agent_help_text(lang)`、`version_text()`；**结果标记**（`[选择的选项]`/`[用户输入]`/`[图片]`/`[文件]`/`[状态]`、`# Qn`、`---`）纳入词条，`agent_help_text` 中对这些标记的描述用同语言、与 `cli/output.rs` 实际输出一致。
- `cli/output.rs`：输出区块标题/标记改用 `tr`。**结构不变**（分组顺序/分隔符 `---`/`# Qn` 形态），仅文案随语言。
- `cli/args.rs`、`cli/mod.rs`、`image_writer.rs`、`file_attachment.rs`：参数/IO 错误文案走 `tr`。
- `prompts.rs::cli_reference()`（D7）：**英文固定**，且**不再内嵌** `agent_help_text`；改为行为约束规则 + 一行执行指引「Run `<prog> --agent-help` to see usage.」。`agent_help_text` 自此仅供 `--agent-help`（本地化）。

### 4.3 窗口标题 / 菜单 / Dock / 通知
- `app/mod.rs`：创建窗口标题用 `tr`。
- `macos_menu.rs`、`macos_dock_icon.rs`：标题/提示用 `tr`。
- 通知（全局搜 `Notification` 确认位置）：文案用 `tr`。

### 4.4 语言切换即时生效（D9）
- `save_settings`：检测 `language` 变化时——重建 macOS 菜单、更新已打开 `popup`/`settings` 窗口标题（dock 提示同理）。`settings-updated` 仍广播给前端切 locale。

## 5. 远程渠道（D8）

- `channels/telegram.rs`、`telegram/*`、`channels/dingding.rs`、`dingtalk/card.rs`：构造发给用户的文案处取 `Lang::resolve(config.language)` 并用 `tr`。
- 已发出的消息不追溯改写；新消息按当前语言。

## 6. 设置页 UI

- 「外观」卡片新增「界面语言」下拉：跟随系统 / English / 简体中文 → 写 `config.general.language`，`persist()` + 本窗口即时切 locale。

## 7. 里程碑

- **M1 基础设施**：`config.language` + `types.ts`；装 `vue-i18n`/`sys-locale`；前端 i18n 框架 + locale 解析 + 实时切换；Rust `i18n.rs` + locale 解析；设置「界面语言」下拉。（此时切换可见但词条尚少）
- **M2 前端 GUI 文案**：抽取 `App`/`Popup`/`Settings` + `shortcut.ts`，补全 `en`/`zh` 词条。
- **M3 语音事件 key 化**：Swift + `speech.rs` 改发 key（+参数），前端 `speech.*` 翻译。
- **M4 Rust 可见文案**：CLI（help/agent-help/version/args/output 标记）+ 标题 + 菜单/Dock + 通知；prompts 固定英文；语言切换即时重建菜单/标题。
- **M5 远程渠道**：Telegram/钉钉 文案本地化。
- **M6 收尾/QA**：auto/en/zh 三态全面回归、回退英文、无裸 key/无中文残留（注释除外）、构建与 CI 无碍。

## 8. 风险与回退

- **AI 解析契约变动（D6）**：结果标记随语言变化。缓解：`--agent-help` 与 `output.rs` 共用同一套词条 key，保证「文档=实际输出」自洽；结构（分组/分隔）保持不变以降低破坏面。
- **菜单/标题实时重建**：Tauri/macOS 重建菜单的可行性在 M4 先验证；不行则退化为「下次启动生效」并回询确认。
- **翻译完整性**：以「无裸 key、无中文残留」为验收闸；fallback=en 兜底。
- **vue-i18n 构建集成**：与现有 Vite/Tauri 构建对齐，避免 SSR/legacy 模式坑（用 `legacy:false`）。
