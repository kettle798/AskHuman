# 需求：项目多语言（i18n，英文为源 + 简体中文）

> 状态：已确认（待按计划实现）
> 关联计划：`docs/plans/i18n.md`

## 1. 背景

当前项目所有用户可见文案均为中文（硬编码在 Vue 视图、Rust CLI/渠道/菜单、Swift 语音状态等处）。需要把项目国际化：**以英文为源语言（开发默认）**，并提供**简体中文**翻译，运行时可切换。

## 2. 目标

- 全部「用户可见」文案支持英文 / 简体中文两套，运行时按配置切换；**代码注释不翻译**。
- 设置页提供「界面语言」选择：跟随系统（默认）/ English / 简体中文；任何缺失/未知一律回退英文。
- 语言为单一配置项，**前端 + 后端 + CLI 共用**；切换后已打开窗口实时生效。

## 3. 国际化范围（D1：全部用户可见文案）

| 区域 | 具体位置 | 处理方式 |
|---|---|---|
| 桌面 GUI | `src/views/PopupView.vue`、`SettingsView.vue`、`App.vue`、`src/lib/shortcut.ts`（冲突提示/“无”）、`src/lib/theme.ts`（可见错误） | vue-i18n `t()` |
| GUI 内的后端文案 | 语音状态/错误（`speech.rs` + Swift `SpeechEngine`/`SpeechBridge` 经 `speech-*` 事件） | 后端发**语义 key（+参数）**，前端用 vue-i18n 翻译 |
| CLI | `cli/help.rs`（`--help`/`--agent-help`/`--version`）、`cli/args.rs`（参数错误）、`cli/output.rs`（结果区块与标记）、`cli/mod.rs`/`image_writer.rs`/`file_attachment.rs`（错误） | Rust i18n 直接产出对应语言 |
| 窗口标题 | `app/mod.rs`（“Question from …”等） | Rust i18n |
| macOS 菜单 / Dock | `macos_menu.rs`、`macos_dock_icon.rs` | Rust i18n，切换语言**立即重建** |
| 系统通知（如有） | 全局搜索 `Notification` 确认 | Rust i18n |
| 远程渠道推送 | `channels/telegram.rs`、`telegram/*`、`channels/dingding.rs`、`dingtalk/card.rs` 等面向用户的文案 | Rust i18n，跟随 app 界面语言 |
| 参考提示词 | `prompts.rs::cli_reference()`（复制给 AI 用） | **始终英文**；**不再内嵌** help 文本，改为提示 AI 执行 `<prog> --agent-help` 获取用法（见 D7） |

## 4. 已确认决策

| 编号 | 决策项 | 结论 |
|---|---|---|
| D1 | 范围 | 全部用户可见文案（见 §3）。源语言=英文 |
| D2 | 前端方案 | **vue-i18n**（Composition API，`t()`，按命名空间组织词条） |
| D3 | 语言项与默认 | 设置「界面语言」下拉：**跟随系统(默认) / English / 简体中文**；回退英文 |
| D4 | GUI 内后端文案 | 后端发**语义 key（+参数）**，前端 vue-i18n 统一翻译（语音状态/错误） |
| D5 | 配置存储 | `config.general.language = "auto" \| "en" \| "zh"`，前后端共用 |
| D6 | CLI 粒度 | **全部本地化**，含 `--agent-help` 与结果标记（`[选择的选项]`/`[用户输入]`/`[图片]`/`[文件]`/`[状态]`、`# Qn` 分组、`---` 分隔）。接受 AI 解析契约随语言变化——`--agent-help` 用同一语言描述这些标记，保持自洽 |
| D7 | 参考提示词 | `cli_reference()` **始终英文**，且**不再内嵌** `agent_help_text`，改为提示 AI「执行 `<prog> --agent-help` 获取使用方法」。故 `agent_help_text` 仅供 `--agent-help`（本地化），prompts 与之解耦 |
| D8 | 远程渠道文案 | 跟随 app 界面语言（`config.language`；`auto`→系统） |
| D9 | macOS 菜单/Dock | 语言切换后**立即重建生效**（同时更新已打开窗口标题） |
| D10 | “跟随系统”判定 | 系统首选语言以 `zh` 开头→中文，否则→英文 |
| D11 | Rust 端 auto 探测 | 引入 **`sys-locale`** crate |
| D12 | 翻译边界 | 仅用户可见字符串；**注释保持原样**；词条 key 用英文命名 |
| D13 | `shortcut.ts` | 冲突提示/“无”改为返回 **key（+参数）**，由 `SettingsView` 用 `t()` 渲染（保持 util 纯净） |

## 5. 约束与既有规则（不可破坏）

- **配置容错**：`config.json` 缺 `language` 字段时回退 `auto`（沿用现有 serde `default` 容错）。
- **实时生效**：沿用现有 `settings-updated` 广播；前端切 locale，后端（菜单/标题）即时重建。
- **跨平台**：`sys-locale` 跨平台；macOS 专属（菜单/Dock/语音）逻辑仍仅 macOS 编译。
- **stdout 契约**：CLI 结果**结构**（分组/分隔/区块顺序）不变，仅其中文案/标记随语言切换（D6）。
- **单文件分发 / release 模式**等既有约束不变。

## 6. 验收标准

1. 设置「界面语言」切 English / 简体中文 / 跟随系统，弹窗与设置窗口**实时**整窗换语言（含语音按钮提示、状态/错误、确认弹层、文件 chip 等）。
2. `config.language=auto` 时：系统中文→中文界面，其它→英文界面；显式 en/zh 覆盖系统。
3. 语音状态/错误按界面语言显示（后端发 key，前端翻译）；机型不支持语言等错误文案正确本地化（含参数）。
4. CLI：`--help`/`--agent-help`/`--version`/参数错误/结果输出（含标记与 `# Qn`/`---`）随语言切换；`--agent-help` 与实际输出标记一致自洽。
5. 参考提示词（设置→集成）**始终英文**。
6. macOS 菜单、Dock、窗口标题随语言切换即时更新；通知（如有）按语言。
7. Telegram/钉钉 推送文案按 app 界面语言。
8. 任意缺失词条回退英文，不出现裸 key；无未翻译的中文残留（注释除外）。
9. 参考提示词不含内嵌 help，而是提示执行 `--agent-help`；按提示执行能拿到当前语言的完整用法。

## 7. 反馈意见

- （2026-06-06）参考提示词调整：**不在提示词里内嵌 help 文本**，改为提示 AI「执行 `<prog> --agent-help` 获取使用方法」。即 `cli_reference()` 与 `agent_help_text` 解耦，prompts 仅保留行为约束规则 + 一行执行指引（英文）。
